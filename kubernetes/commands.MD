# Here are some commonly used kubectl commands in Kubernetes:

kubectl create: Creates a resource from a file or standard input.

kubectl apply: Applies a configuration to a resource by filename, directory, or stdin.

kubectl get: Displays information about one or more resources.

kubectl describe: Shows detailed information about a resource.

kubectl logs: Prints the logs for a container in a pod.

kubectl exec: Runs a command in a container in a pod.

kubectl port-forward: Forwards one or more local ports to a pod.

kubectl delete: Deletes one or more resources.

kubectl scale: Scales a deployment, replica set, or replication controller.

kubectl rollout: Manages rollouts of a deployment.

kubectl config: Manages the kubeconfig file.

kubectl version: Displays the version of the kubectl client and the Kubernetes server.
